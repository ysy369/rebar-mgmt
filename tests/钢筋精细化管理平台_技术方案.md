# 钢筋精细化管理平台技术方案

---

## 一、项目概述

### 1.1 建设背景

随着建筑工程项目规模扩大与管理精细化要求提升，传统基于 WPS Excel + VBA 的钢筋数据管理模式已难以满足多项目协同、实时数据汇总及成本动态管控的需求。现场精管工程师分散于各工地，数据填报格式不一、汇总效率低下、成本分析滞后，导致管理层无法及时掌握项目钢筋使用情况及结余率。

本平台旨在通过信息化手段，实现钢筋数据从现场采集、自动汇总到成本分析的全流程数字化管理，提升数据准确性、汇总效率及成本管控能力。

### 1.2 建设目标

本平台以 **"数据驱动、闭环管控、降本增效"** 为核心理念，旨在提供一套覆盖钢筋预算、翻样、进场、消耗、盘点、结算全过程的数字化管理工具：

- **一屏统览**：通过可视化数据看板实时展示项目钢筋管理效益、损耗率、结余率等核心 KPI；
- **过程留痕**：实现进场、调拨、盘点、废料等关键业务数据的在线录入与自动汇总，全程操作日志可追溯；
- **自动计算**：后端计算引擎基于国家标准公式自动计算理论库存、实际库存、盈亏量、损耗率、效益率；
- **预警纠偏**：对异常指标自动触发预警通知，帮助管理层及时发现并纠正管理偏差；
- **辅助决策**：为月度结算、供应商评价、劳务考核提供数据支撑，支持多项目横向对比分析。

## 二、系统架构与部署

### 2.1 总体架构

系统采用 **B/S 三层架构**（表现层 / 业务逻辑层 / 数据访问层），遵循 MVC 设计模式，各层职责清晰、松耦合，便于维护和扩展。用户无需安装客户端，通过浏览器即可访问：

| 层级 | 核心职责 | 关键技术能力 |
|------|---------|-------------|
| **表现层** | 浏览器端页面渲染与交互 | Bootstrap 5 响应式布局、Chart.js 数据可视化引擎、AJAX 异步数据交互、RESTful 风格 API 调用 |
| **业务层** | 平台核心逻辑处理 | Flask Web 应用框架、Blueprint 模块化路由、SQLAlchemy ORM 映射层、用户认证与 RBAC 鉴权、Excel 导入导出引擎（openpyxl / pandas）、附件与图片管理服务 |
| **数据层** | 数据持久化与文件存储 | MySQL 8.0 关系型数据库（InnoDB 事务引擎、UTF-8 字符集）、文件系统存储（项目附件 / 现场图片）、操作日志与审计记录 |

**数据层特别说明**：除结构化台账数据外，平台还具备完善的文件与图片管理能力，支持项目现场照片、效果图、单据扫描件等附件的上传、存储、预览与管理，为甲方提供直观的项目可视化档案。

### 2.2 部署说明

- **部署环境**：Windows 云服务器，支持 Docker 容器化部署（Docker Compose 一键编排 MySQL + 应用服务），降低环境依赖与运维复杂度；
- **访问方式**：浏览器直接访问，支持 PC、平板等终端，无需安装任何客户端，兼容主流浏览器（Chrome / Edge / Firefox）；
- **网络要求**：内网或 VPN 环境即可使用，满足施工现场多点位协同，支持并发用户同时在线操作；
- **数据安全**：采用 bcrypt 密码哈希算法存储凭据、CSRF 跨站请求伪造防护、Session 会话管理、角色分级授权、操作日志全程留痕，确保数据安全与可追溯；
- **扩展能力**：支持后续按项目、按甲方单位横向扩展，满足集团化管理需求；架构预留数据接口，可对接企业 ERP 或财务系统。

## 三、功能模块详细设计

### 3.1 数据看板（首页）

数据看板是掌握全局的核心入口，默认展示**全期指标**，并支持按月份动态追加月度指标。页面采用 AJAX 异步加载，筛选切换时无需整页刷新，交互体验流畅。

#### 3.1.1 顶部深色 KPI 栏

集中展示五大核心经营指标，并附带状态标签（动态语义化着色——绿色正常、黄色关注、红色预警），直观反映管理健康度：

| 指标 | 说明 | 状态标签示例 |
|------|------|-------------|
| 管理效益（T） | 全期累计节约钢筋量 | 节约 / 持平 |
| 损耗率（%） | 废料量占进场量比例 | 正常 / 关注 / 偏高 |
| 效益率（%） | 管理效益占预算总量比例 | 优秀 / 良好 / 亏损 |
| 预算总量（T） | 对甲结算量/施工图预算量 | 对甲结算 |
| 翻样总量（T） | 现场钢筋翻样量 | 累计翻样 |
| 进场总量（T） | 累计进场钢筋量 | 累计进场 |

#### 3.1.2 快捷指标卡片

以四张等宽卡片展示关键经营数据，附带小标签（"全期累计""累计构件""效益率""累计进场"），便于管理层快速抓取重点：

- **预算总量**：全期累计预算量；
- **翻样总量**：累计翻样构件数及吨数；
- **消耗总量**：全期累计消耗量，附效益率标签；
- **进场总量**：全期累计进场量。

#### 3.1.3 筛选条件

- **统计周期**：支持"全期"或选择具体月份，选择月份后页面通过 AJAX 动态追加月度指标区块，无需刷新页面；
- **承接公司**：按承接单位筛选项目，下拉列表通过级联接口动态加载；
- **甲方单位**：按甲方单位筛选项目；
- **重置按钮**：一键恢复默认全期视图。

#### 3.1.4 业务模块入口

提供 8 个常用功能入口，以图标卡片矩阵呈现，涵盖钢筋管理全业务流程：钢筋预算、钢筋翻样、钢筋物资、策划管理、巡检台帐、盈亏分析、结算管理、分包领用。

#### 3.1.5 主要指数图表

采用 Chart.js 柱状图（Bar Chart）对比展示预算、消耗、节约等核心指标，支持鼠标悬停查看数据明细。全期视图下直接展示项目维度数据；选择月份后，右上角出现"本期/全期"切换按钮，默认展示当月数据，可切换查看全期对比。

#### 3.1.6 指数预警模块

后端定时扫描各项目数据，对结余率 < 设定保底或目标结余率（亏损）、消耗量 > 预算量（超耗）、进场量 = 0（无记录）等异常情况生成预警卡片并向管理人员示警，以不同颜色和图标区分预警等级。无异常时展示"暂无异常指标"，帮助管理人员快速定位问题项目。

#### 3.1.7 耗用指数图表

以 Chart.js 折线图（Line Chart）展示消耗量月度趋势，支持按"施工进度 / 翻样进度 / 采购进度"三个维度切换数据源，辅助管理层判断工程推进节奏与钢筋供应的匹配程度。

#### 3.1.8 项目数据汇总表

采用服务端分页（Server-side Pagination）展示所有项目关键数据，避免大数据量下前端性能瓶颈。列包括：项目名称、甲方、承接公司、对甲结算量、使用量、节约量、结余率，每行支持点击查看详情。

### 3.2 钢筋台账管理

钢筋台账是平台的底层数据入口，覆盖钢筋全生命周期管理，每个台账模块均支持 Excel 模板批量导入（openpyxl 解析引擎）与条件筛选查询：

- **进场台账**：记录每批次钢筋的品牌、规格（直径 / 长度）、理论重量、过磅重量、进场日期、车牌号、使用部位、劳务队等信息，支持毛重/皮重/净重自动换算；
- **调拨台账**：记录项目间钢筋调入/调出数据，区分原材料/半成品调拨类型，追踪调入方与调出方；
- **措施筋台账**：记录措施筋使用部位、规格（HRB400 / HPB300）、重量，区分预算内/非预算收入类别，关联签证单号；
- **盘点台账**：记录原材、短料、半成品各规格的实际盘点量，与理论库存自动对比生成盈亏量；
- **废料台账**：记录废料处理日期、车号（处理前/后）、废料重量、钢筋长度、所属劳务队等信息；
- **形象进度**：记录楼栋、楼层施工进度状态（未施工/施工中/已完成），关联模型量数据，支持逐层细化。

所有台账导入后自动触发后端汇总计算，实时更新项目分析指标，无需人工干预。

### 3.3 项目管理

- **项目管理**：创建、编辑、归档项目，关联甲方单位和承接公司，支持项目状态流转（活跃/已归档）；
- **甲方单位管理**：维护甲方单位信息，支持跨项目成本对比与经营分析；
- **承接公司管理**：维护分包/承接单位信息（联系人、电话、地址），为供应商评价提供数据基础；
- **项目附件管理**：支持上传项目效果图、现场照片、合同扫描件等附件，自动识别文件类型与大小，支持按类型分类浏览与设置封面图；
- **项目授权**：为普通用户分配可操作的项目（多对多关联），确保数据严格按项目隔离。

### 3.4 审核与分析

- **料单审核**：钢筋配料单在线提交、多级审核（草稿→已提交→已审核→已批准/已驳回），驳回后可重新编辑提交，全程留存审核记录与操作日志；
- **批次审核**：对 Excel 导入批次进行数据质量审核，展示成功/失败行数及错误明细，支持批量通过或驳回；
- **盈亏分析**：从进场量、调拨量、措施筋量、盘点量、料单审核、项目成本等多维度综合分析项目盈亏，提供详细数据明细与指标钻取。

### 3.5 系统设置

- **用户管理**：创建、启用/禁用用户，分配角色和项目权限，记录最后登录时间；
- **操作日志**：记录关键操作（创建/编辑/删除/导入/导出），包含操作人、目标对象、IP 地址、时间戳，便于审计与追溯。

## 四、核心数据模型

平台围绕"项目"建立数据模型，基于 SQLAlchemy ORM 进行对象关系映射，核心数据流为：**预算 → 翻样 → 进场 → 消耗 → 盘点 → 结算**。

### 4.1 项目分析汇总表（project_analysis）

该表存储各项目的核心经营指标，是数据看板、盈亏分析的数据来源，由后端计算引擎根据台账原始数据自动汇总生成：

| 数据项 | 业务含义 | 计算来源 |
|------|---------|---------|
| 施工图预算量 | 施工图纸核算的钢筋总用量 | 主体结构量 + 措施筋量 + 非实体量 - 二构量 |
| 翻样总量 | 经翻样细化后的构件级钢筋用量 | BOM 配料单汇总 |
| 消耗总量 | 现场实际消耗的钢筋总量 | 各台账消耗量汇总 |
| 进场总量 | 累计进场入库的钢筋量 | 进场台账汇总 |
| 非预算用量 | 非预算收入的钢筋使用量 | 措施筋台账（非预算类）汇总 |
| 临时设施用量 | 临建设施消耗的钢筋量 | 措施筋台账（临建类）汇总 |
| 建筑面积 | 项目总建筑面积（㎡） | 手动录入 |
| 钢筋含量 | 单位建筑面积钢筋用量（kg/㎡） | 预算量 / 建筑面积 |
| 节约量 | 预算量与实际使用量的差值 | 预算量 - 使用量 |
| 结余率 | 节约量占预算量的比例 | 节约量 / 预算量 × 100% |
| 损耗率 | 废料量占进场量的比例 | 废料量 / 进场量 × 100% |
| 效益率 | 节约量占预算总量的比例 | 节约量 / 预算总量 × 100% |

### 4.2 核心台账数据

各台账表通过外键（Foreign Key）关联项目，配合数据库索引优化高频查询性能：

| 数据类别 | 业务含义 | 关键字段 |
|------|---------|---------|
| 进场台账 | 批次级进场记录 | 日期、品牌、规格、理论重量、过磅重量、车牌号、使用部位 |
| 调拨台账 | 项目间调拨记录 | 日期、调拨方向（入/出）、规格、根数、重量、调入项目 |
| 措施筋台账 | 措施筋使用记录 | 日期、规格（HRB400/HPB300）、重量、预算/非预算类别 |
| 盘点台账 | 定期库存盘点 | 盘点日期、类别（原材/短料/半成品）、规格、实际重量 |
| 废料台账 | 废料处理记录 | 日期、车号、废料重量、钢筋长度、劳务队 |
| 模型量台账 | 主体结构模型量 | 日期、主体结构重量、二构重量 |

### 4.3 项目附件与图片

平台内置附件管理能力，基于文件系统存储、数据库记录元信息的混合架构，支持项目相关文件与图片的存储与管理：

| 数据项 | 业务含义 |

### 4.4 权限相关数据

| 数据类别 | 业务含义 | 关联关系 |
|------|---------|---------|

## 五、关键业务流程

### 5.1 预算 → 翻样 → 进场 → 消耗闭环

钢筋管理全生命周期数据流如下，平台在各节点采集原始数据后，通过后端计算引擎自动串联形成完整数据链：

```
施工图预算量（基于施工图纸导入或手动录入）
    ↓
钢筋翻样量（BOM 配料单细化到构件级别，支持 Excel 批量导入）
    ↓
进场量（实际采购/进场过磅，记录理论重量与过磅重量双向校验）
    ↓
消耗量（根据进场、盘点、废料等数据反向推算实际使用量）
    ↓
盘点量（现场实际剩余量，与理论库存对比生成盈亏）
    ↓
废料量（废料处理量，计入损耗率计算）
    ↓
结算量（对甲结算依据，结合合同单价自动计算结算金额）
```

平台通过对上述各环节数据的在线采集与自动汇总计算，形成钢筋管理的完整数据链，确保每个环节有据可查、量值可追溯、异常可预警。

### 5.2 盈亏计算逻辑

#### 5.2.1 核心公式

**施工图预算量**：

```
施工图预算量 = 主体结构量 + 措施筋量 + 非实体量 - 二构量
```

**使用量**（从进场到实际消耗的综合推算）：

```
使用量 = 进场量 - 剩余量 - 废料量 - 已施工二构量 - 调拨量
       - 非预算用量 - 临时设施用量
```

**节约量**（正值为节约，负值为超耗）：

```
节约量 = 施工图预算量 - 使用量
```

**结余率**（衡量预算执行效率）：

```
结余率 = 节约量 ÷ 施工图预算量 × 100%
```

#### 5.2.2 理论库存与实际库存

**理论库存**（基于出入库流水推算）：

```
理论库存 = 上期库存 + 本期进场 + 本期调入
          - 本期消耗 - 本期调出 - 本期废料
```

**盈亏量**（账实差异分析）：

```
盈亏量 = 实际库存 - 理论库存
```

- 盈亏量为正：现场实际库存多于理论库存，可能存在进场未入账或消耗少报；
- 盈亏量为负：现场实际库存少于理论库存，可能存在浪费、丢失或未入账出库。

#### 5.2.3 损耗率与效益率

**损耗率**（衡量现场钢筋利用率）：

```
损耗率 = 废料量 ÷ 进场量 × 100%
```

- 损耗率越低，说明现场钢筋利用率越高，管理越精细；
- 行业平均水平约 2%–4%，超过 5% 触发系统自动预警。

**效益率**（衡量钢筋管理综合效益）：

```
效益率 = 节约量 ÷ 预算总量 × 100%
```

- 效益率为正且越高，说明项目钢筋管理效益越好；
- 为负则提示项目已超预算，需立即采取纠偏措施。

## 六、权限体系

平台采用基于角色的访问控制模型（RBAC — Role-Based Access Control），结合 Flask-Login 会话认证与 bcrypt 密码哈希，确保数据安全与业务隔离：

| 角色 | 权限范围 | 适用对象 |
|------|---------|---------|
| 系统管理员 | 全部项目数据、用户管理、系统配置、跨项目对比分析 | 公司信息化负责人 / 高层管理者 |
| 甲方管理员 | 所辖甲方单位下全部项目数据 | 甲方项目管理人员 |
| 审核员 | 料单审核、批次审核、数据校验 | 甲方技术 / 成本控制人员 |
| 查看员 | 仅查看被授权项目的数据看板与报表（只读权限） | 甲方现场代表、监理工程师 |

**数据隔离原则**：普通用户只能查看和操作被授权项目的数据，项目之间数据严格隔离（基于 user_projects 多对多关联表控制）；管理员可查看全部项目并进行跨项目对比分析。

**安全机制**：

- 密码采用 bcrypt 算法哈希存储（12 轮加密），不可逆；
- 所有 POST 请求启用 CSRF Token 校验，防止跨站请求伪造；
- Session 会话设置 HttpOnly + SameSite=Lax，防止 XSS 窃取；
- 关键操作（创建/编辑/删除/导入/审核）均有操作日志记录，包含操作人、目标对象、IP 地址、时间戳，满足审计合规要求。
