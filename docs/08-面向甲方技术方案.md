# 钢筋精细化管理平台 — 技术方案

> 版本：v1.0 | 日期：2026-06-22 | 密级：项目机密

---

## 一、方案概述

### 1.1 项目背景

当前建筑行业钢筋管理普遍依赖人工 Excel 台账，存在以下痛点：

| 痛点 | 现状 | 影响 |
|------|------|------|
| **数据分散** | 进场、调拨、措施筋、盘点等数据分散在不同 Excel 文件中，版本不一 | 查找困难，历史数据易丢失 |
| **汇总滞后** | 月末手工汇总，公式易出错，不同项目口径不统一 | 决策滞后，可信度低 |
| **成本核算不准** | 结余率靠人工估算，缺少系统化计算 | 利润分析偏差大 |
| **权限失控** | Excel 文件可随意复制、修改 | 数据安全无保障 |
| **跨项目难对比** | 管理员需逐个打开各项目 Excel | 管理效率低下 |

本方案构建一套 **B/S 架构的钢筋精细化管理平台**，实现数据集中管理、自动计算、可视化分析和一键导出。

### 1.2 核心价值

```
            ┌─── 数据统一 ───┐
            │  6 类台账集中   │
            │  云端安全存储   │
            └───────┬───────┘
                    │
    ┌─── 计算自动化 ──┼─── 权限精细化 ───┐
    │  13 项指标      │   角色隔离        │
    │  秒级汇总        │   项目级授权      │
    └───────┬───────┘    └───────┬───────┘
            │                    │
            └────────┬───────────┘
                     │
              ┌──────┴──────┐
              │  降本增效    │
              │  管理升级    │
              └─────────────┘
```

## 二、系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    用户浏览器 (Chrome / Edge)                  │
│                 https://服务器地址:5000                        │
└──────────────────────────┬──────────────────────────────────┘
                           │  HTTP / HTTPS
┌──────────────────────────┴──────────────────────────────────┐
│                    Nginx (可选反向代理)                        │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────┴──────────────────────────────────┐
│                 Docker 容器编排 (双容器)                       │
│  ┌─────────────────────┐    ┌─────────────────────┐         │
│  │  rebar-app           │    │  rebar-db            │         │
│  │  Flask + Waitress    │◄──►│  MySQL 8.0           │         │
│  │  端口 5000 (对外)    │    │  端口 3306 (内部)    │         │
│  └─────────────────────┘    └─────────────────────┘         │
└─────────────────────────────────────────────────────────────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        uploads/      exports/     mysql_data/
        (上传文件)    (导出文件)    (数据库持久化)
```

### 2.2 技术选型

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| **Web 框架** | Flask | 3.x | 轻量灵活，学习曲线低，适合中小规模应用 |
| **数据库** | MySQL | 8.0+ | Windows 运行稳定，团队运维经验丰富 |
| **ORM** | SQLAlchemy | 3.x | 参数化查询防注入，开发效率高 |
| **Excel 解析** | pandas + openpyxl | 2.x / 3.x | pandas 批量处理高效，openpyxl 支持公式写入 |
| **密码安全** | bcrypt | 4.x | 12 轮迭代哈希，行业标准 |
| **前端框架** | Bootstrap 5 | 5.x | 专业 UI 组件，响应式布局，无构建依赖 |
| **图表** | Chart.js | 4.x | 轻量可视化，支持趋势图、柱状图、雷达图 |
| **容器化** | Docker Compose | — | 一键部署，环境隔离，无需安装 Python/MySQL |
| **生产服务器** | Waitress | 3.x | Windows 原生 WSGI，纯 Python 无 C 依赖 |

### 2.3 安全设计

| 层面 | 措施 | 说明 |
|------|------|------|
| **身份认证** | Flask-Login + Session | 8 小时超时，HttpOnly Cookie |
| **密码安全** | bcrypt (12 轮) | 不可逆哈希存储，无法反推 |
| **SQL 注入防护** | SQLAlchemy ORM | 100% 参数化查询 |
| **XSS 防护** | Jinja2 自动转义 | 所有用户输入 HTML 转义 |
| **权限控制** | 服务端逐请求验证 | 不依赖前端，无法绕过 |
| **文件安全** | 仅允许 .xlsx，校验文件头 | 防止恶意文件上传 |
| **数据隔离** | 项目级权限过滤 | 普通用户只能访问授权项目 |

### 3.3 核心业务模块

#### 3.3.1 钢筋台账管理（6 类核心数据）

| 序号 | 台账类型 | 数据说明 | 关键字段 | 对应 Excel 模板 |
|:----:|----------|----------|----------|-----------------|
| 1 | **进场量** | 钢筋进场确认 | 进场日期、规格、理论重量、过磅重量、车牌号 | 钢筋进场确认单 |
| 2 | **调拨量** | 钢筋调拨记录 | 调拨日期、类型(原材/半成品)、方向、调入项目 | 调拨确认单 |
| 3 | **措施筋** | 措施筋使用台账 | 用途(马凳筋/梯子筋等)、规格、预算内外分类 | 措施筋台账 |
| 4 | **盘点量** | 库存盘点记录 | 盘点日期、类别(原材/短料/半成品)、规格、重量 | 钢筋盘点表 |
| 5 | **废料量** | 废料处理台账 | 处理日期、过磅重量、劳务队、车牌号 | 废料处理台账 |
| 6 | **形象进度** | 施工进度确认 | 楼栋、楼层、构件类型、施工状态 | 形象进度确认单 |

#### 3.3.2 配料单（BOM）管理

基于真实 Excel 配料单，支持**四层级结构**自动解析：

```
项目 → 楼栋 → 楼层 → 区域 → 构件 → 钢筋明细
  │      │      │      │      │
  │      │      │      │      └── 构件类型：柱/梁/板/墙/楼梯/其他
  │      │      │      └── 施工状态：未开始/施工中/已浇筑
  │      │      └── 区域名称
  │      └── 楼层名称（自动排序）
  └── 楼栋名称（如 1#楼、2#楼）
```

**核心能力**：
- 配料单 Excel 批量导入，自动创建楼栋/楼层/区域/构件层级
- 按直径（6/8/10/12/14/16/18/20/22/25/28/32mm）汇总重量
- 构件施工状态跟踪（未开始 → 施工中 → 已浇筑）
- 管理员审核流程（待审核 → 审核通过/驳回）

#### 3.3.3 料单（切割单）管理

### 3.4 智能分析引擎

#### 3.4.1 结余率 13 项指标体系

基于钢筋管理行业标准公式，系统自动从 6 张子表聚合数据，计算 13 项指标：

```
                            结余率计算模型

┌──────────────────────────────────────────────────────────┐
│                                                          │
│   施工图预算量 ⑤                                           │
│   ┌──────────────────────────────────────────────┐       │
│   │ 主体结构量① + 措施筋② + 非实体量③ - 二构量④  │       │
│   └──────────────────────────────────────────────┘       │
│                           ─                              │
│   实际使用量 ⑪                                             │
│   ┌──────────────────────────────────────────────────┐   │
│   │ 进场量⑥ - 剩余量⑦ - 废料量⑧ - 已施工二构量⑨ - 调拨量⑩ │   │
│   └──────────────────────────────────────────────────┘   │
│                           ═                              │
│   节约量 ⑫ = ⑤ - ⑪                                       │
│   结余率 ⑬ = ⑫ ÷ ⑤ × 100%                                 │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

| 指标编号 | 指标名称 | 数据来源 |
|:--------:|----------|----------|
| ① | 主体结构量 | 模型量表 |
| ② | 措施筋 | 措施筋台账 |
| ③ | 非实体量 | 措施筋-非预算收入 |
| ④ | 二构量 | 模型量表 |
| ⑤ | 施工图预算量 | ①+②+③-④（自动计算） |
| ⑥ | 进场量 | 进场台账 |
| ⑦ | 剩余量 | 盘点表 |
| ⑧ | 废料量 | 废料处理台账 |
| ⑨ | 已施工二构量 | 形象进度确认单 |
| ⑩ | 调拨量 | 调拨确认单 |
| ⑪ | 使用量 | ⑥-⑦-⑧-⑨-⑩（自动计算） |
| ⑫ | 节约量 | ⑤-⑪（自动计算） |
| ⑬ | 结余率 | ⑫÷⑤×100%（自动计算） |

#### 3.4.2 项目盈亏分析

```
    钢筋节约量 × 节约定价  =  节超收益
              +
          其他收入
              =
          总收入
              -
    总成本（6 类费用汇总）
              =
          净利润
            
    利润率 = 净利润 ÷ 总收入 × 100%
```

#### 3.4.3 预警指数系统

管理员看板内置智能预警：

| 预警指标 | 计算方式 | 预警阈值 |
|----------|----------|----------|
| 损耗率 | 废料量 ÷ 进场量 | > 3% |
| 效益率 | 节约量 ÷ 预算量 | < 0% |
| 结余率偏离 | 当前值 vs 目标值 | < 目标值 |

### 3.6 Excel 导入 / 导出

#### 3.6.1 导入能力

```
Excel 上传 → 格式校验 → 字段映射 → 数据校验 → 事务入库 → 自动汇总
                │                       │
                └── 格式错误 → 返回提示   └── 数据错误 → 详细行级错误报告
```

**核心特性**：
- 支持 WPS / Microsoft Excel 的 .xlsx 格式
- 6 种业务模板自动识别和解析
- 必填字段检查、数据类型校验、数值范围验证
- 校验失败显示**逐行错误详情**，指导修正
- 数据库事务保护，失败自动回滚
- 完整的导入历史记录（操作人、时间、成功/失败行数）

#### 3.6.2 导出能力

- 单项目汇总表导出（保留结余率公式）
- 管理员全局汇总导出（跨项目）
- 导出文件保留超链接（汇总 ⇄ 明细可跳转）
- 保留 Excel 公式，导出后仍可计算
- 导出文件保留专业样式排版

## 四、部署方案

### 4.1 推荐方案：Docker Compose 一键部署

```bash
# 唯一前置条件：安装 Docker Desktop（Windows）
# 启动系统（首次自动构建 + 初始化数据库）
docker compose up -d

# 操作完成。浏览器访问 http://服务器IP:5000
```

**优势**：
- ✅ 无需在服务器安装 Python、MySQL
- ✅ 环境完全隔离，不污染宿主机
- ✅ 一条命令启动/停止
- ✅ 数据通过卷持久化，重启不丢失
- ✅ 数据库端口不对外暴露，安全

### 4.2 服务器要求

| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| 操作系统 | Windows Server 2016+ / Windows 10+ | Windows Server 2022 |
| CPU | 2 核 | 4 核+ |
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 50 GB | 100 GB+ (SSD) |
| 网络 | 固定内网 IP | 固定内网 IP |

## 六、技术亮点

### 6.1 真实模板驱动设计

系统设计全程基于 8 个**真实 Excel 模板**反向推导，确保字段映射与用户现有工作流程完全一致，减少使用摩擦。

### 6.2 行业公式内建

结余率 13 项指标、损耗率、效益率等钢筋管理专业公式已内建在系统中，Excel 导入后自动计算，无需人工干预。

### 6.3 分层架构

```
Templates (视图层)
    │
Routes (路由层) ─── 仅做参数接收和返回
    │
Services (服务层) ─── 全部业务逻辑
    │
Models (数据层) ─── SQLAlchemy ORM
```

- 职责分离清晰，维护成本低
- 服务层可独立测试
- 数据库切换只需修改配置

### 6.4 操作可审计

所有关键操作（导入、审核、用户管理）自动记录到 `operation_logs` 表，包含操作人、时间、IP 地址、操作详情，满足管理审计要求。

### 6.5 兼容性

- **浏览器**：Chrome 80+ / Edge 80+，无需安装插件
- **Excel**：兼容 WPS Office 和 Microsoft Excel
- **部署**：支持 Docker / Windows 原生两种模式
- **设备**：响应式布局，支持平板访问
