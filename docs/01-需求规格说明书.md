# 需求规格说明书

> 版本：v1.0 | 日期：2026-06-21 | 状态：初稿

---

## 1. 项目背景

建筑行业钢筋管理长期依赖人工 Excel 台账，存在数据分散、汇总滞后、成本核算不准确等问题。本项目构建一套 B/S 架构的钢筋精细化管理平台，实现数据集中管理、自动计算、可视化分析和一键导出。

## 2. 用户角色

### 2.1 管理员（admin）
公司管理层，拥有系统最高权限：
- 查看、管理**全部项目**数据
- 导入任意项目的 Excel 数据
- 系统配置（用户管理、项目设置、模板管理）
- 跨项目成本对比分析（可按甲方单位筛选聚合）
- 一键导出任意项目或全局汇总 Excel

### 2.2 普通用户（user）
项目现场精管工程师，权限受限：
- 仅能操作**被授权的项目**（支持一个用户关联多个项目）
- 数据按项目严格隔离，用户 A 看不到用户 B 的项目数据
- 可导入授权项目的 Excel 数据
- 查看授权项目的报表和可视化图表
- 导出授权项目的汇总 Excel

## 3. 功能需求

### 3.1 用户认证模块
- **FR-01**：用户登录（账号+密码）
- **FR-02**：用户登出
- **FR-03**：管理员可创建/启用/禁用用户
- **FR-04**：密码修改

### 3.2 项目管理模块
- **FR-05**：管理员创建/编辑/删除项目
- **FR-06**：项目关联甲方单位（标签属性）
- **FR-07**：管理员为用户分配项目权限
- **FR-08**：普通用户登录后看到自己的项目列表

### 3.3 数据导入模块
- **FR-09**：支持上传 WPS Excel 模板（.xlsx 格式）
- **FR-10**：进场量数据导入（模板 A）
- **FR-11**：调拨量数据导入（模板 B）
- **FR-12**：措施筋数据导入（模板 C）
- **FR-13**：盘点量数据导入（模板 D）
- **FR-14**：导入时数据校验（必填字段、数据类型、数值范围）
- **FR-15**：校验失败时显示详细错误信息，支持重新上传
- **FR-16**：导入成功后数据自动汇总入库
- **FR-17**：支持查看导入历史记录

### 3.4 报表分析模块（基于真实模板公式）

**结余率完整公式**：

```
施工图预算量 ⑤ = 主体结构量① + 措施筋② + 非实体量③ - 二构量④
使用量     ⑪ = 进场量⑥ - 剩余量⑦ - 废料量⑧ - 已施工二构量⑨ - 调拨量⑩
节约量     ⑫ = ⑤ - ⑪
结余率     ⑬ = ⑫ ÷ ⑤ × 100%
```

**数据来源映射（6 张子表）**：

| 指标 | 数据源 | 导入模板 |
|------|--------|---------|
| ① 主体结构量 | 表1模型量 | 施工图预算数据 |
| ② 措施筋 | 表2措施量 / 措施筋台账 | 措施筋台账及申请表 |
| ③ 非实体量 | 措施筋非预算收入 | 非预算收入使用钢筋明细 |
| ④ 二构量 | 二次结构 | 模型量表 |
| ⑥ 进场量 | 表3进场量 / 钢筋进场台账 | 钢筋进场台账 |
| ⑦ 剩余量 | 表4剩余量 / 盘点表 | 钢筋盘点表 |
| ⑧ 废料量 | 表5废料量 / 废料处理台账 | 废料处理台账 |
| ⑨ 已施工二构量 | 进度确认 | 形象进度确认单 |
| ⑩ 调拨量 | 表6调拨量（原材/半成品） | 调拨确认单 |

- **FR-21**：趋势图（折线图）：展示项目各阶段的结余率变化趋势
- **FR-22**：对比柱状图：展示各分项（①~⑩）的用量对比
- **FR-23**：管理员可按甲方单位筛选，聚合查看跨项目成本对比
- **FR-24**：数据明细表（分页、可排序、可筛选）

### 3.5 Excel 导出模块
- **FR-25**：一键导出项目汇总表
- **FR-26**：导出文件保留超链接（可点击跳转到明细）
- **FR-27**：导出文件保留分表计算逻辑（公式不丢失）
- **FR-28**：管理员可导出全局汇总（所有项目）

### 3.6 系统管理模块
- **FR-29**：管理员可管理系统配置参数
- **FR-30**：操作日志记录（谁在何时做了什么）

## 4. 非功能需求

### 4.1 性能
- 页面响应时间 < 2 秒
- Excel 导入（1000 行以内）处理时间 < 10 秒
- 支持 50 个并发用户

### 4.2 安全
- 密码使用 bcrypt 哈希存储
- 所有数据库操作防 SQL 注入
- 权限验证在服务端执行，不依赖前端
- Session 超时自动登出（默认 8 小时）

### 4.3 可用性
- 界面简洁专业，适合非技术人员操作
- Excel 导入错误提示清晰，指导用户修正
- 关键操作有确认提示

### 4.4 可维护性
- 代码遵循 PEP 8 规范
- 核心逻辑与路由分离
- 配置文件独立，不硬编码

## 5. 约束条件

- 部署环境：Windows Server（具体版本待确认）
- 浏览器支持：Chrome 80+、Edge 80+
- Excel 格式：仅支持 .xlsx（WPS Office / Microsoft Excel）
- 数据库：MySQL 8.0+
- Python 版本：3.9+
