> 版本：v1.0 | 日期：2026-06-24

## 一、迁移总览

```
┌─────────────────────────┐         ┌─────────────────────────────┐
│      本地开发机           │         │      云服务器 (Windows)       │
│                         │         │                             │
│  · 源码 + 文档           │  迁移    │  · Docker Compose 运行       │
│  · tests/ Excel 模板     │ ──────→ │  · MySQL 8.0 容器            │
│  · dev-log/ 开发日志     │         │  · Flask + Waitress 容器      │
│                         │         │  · 数据持久化到宿主机          │
└─────────────────────────┘         └─────────────────────────────┘

  迁移方式选择：

  ┌──────────────────────────────────────────────────────────────┐
  │  方式一：Docker Compose（推荐）→ 第 2 章                      │
  │  前提：服务器已安装 Docker Desktop                            │
  │  优点：一键部署，环境隔离，无需安装 Python/MySQL               │
  │                                                              │
  │  方式二：Windows 原生部署 → 第 3 章                            │
  │  前提：服务器已安装 Python 3.9+、MySQL 8.0+                    │
  │  优点：无 Docker 依赖，直接使用 Windows Service                │
  └──────────────────────────────────────────────────────────────┘
```

## 二、方式一：Docker Compose 部署（推荐）

### 2.1 服务器环境准备

在云服务器上完成以下一次性准备工作：

```powershell
# 1. 安装 Docker Desktop for Windows
#    下载地址：https://docs.docker.com/desktop/setup/install/windows-install/
#    安装后重启服务器

# 2. 验证安装
docker --version          # 应显示 Docker version 24.x+
docker compose version    # 应显示 Docker Compose version v2.x+

# 3. （可选）配置 Docker 开机自启
#     Docker Desktop → Settings → General → Start Docker Desktop when you log in
```

### 2.2 上传项目文件

将项目打包上传到云服务器，**以下文件不需要上传**（容器内重新构建）：

```
✅ 需要上传的：
   ├── Dockerfile
   ├── docker-compose.yml
   ├── .env                    ← 需要创建（基于 .env.example 修改）
   ├── requirements.txt
   ├── config.py
   ├── run.py
   ├── app/                    ← 全部应用代码
   ├── docker-entrypoint-initdb.d/  ← 数据库初始化 SQL
   ├── uploads/                ← 如果有已上传的文件
   └── exports/                ← 如果有已导出的文件

❌ 不需要上传的：
   ├── venv/                   ← 虚拟环境（容器内重建）
   ├── __pycache__/            ← Python 缓存
   ├── tests/                  ← 测试 Excel 文件（可选）
   ├── dev-log/                ← 开发日志（可选）
   ├── docs/                   ← 文档（可选）
   └── .vscode/ .idea/         ← IDE 配置
```

#### 方式 A：直接压缩上传

```powershell
# 在本地，压缩项目（排除不需要的文件）
# 用 7-Zip 或 Windows 自带压缩，选择项目根目录
# 上传到服务器后解压到 C:\rebar-mgmt\
```

#### 方式 B：使用 Git（推荐）

```powershell
# 在云服务器上
git clone <你的仓库地址> C:\rebar-mgmt
cd C:\rebar-mgmt
```

### 2.3 配置环境变量

```powershell
# 在服务器上，进入项目目录
cd C:\rebar-mgmt

# 复制环境变量模板
copy .env.example .env

# 编辑 .env（用记事本或 VS Code），修改以下三项为随机值：
#   SECRET_KEY=<生成一个随机字符串>
#   MYSQL_ROOT_PASSWORD=<强密码>
#   MYSQL_PASSWORD=<强密码>
```

**生成安全密钥的方法**：

```powershell
# 在 PowerShell 中运行，生成随机字符串作为 SECRET_KEY
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }))
```

修改后的 `.env` 示例：

```env
SECRET_KEY=a7Kf9mPx2QvL5nR8wY3bE6cH1jU4tG0o
MYSQL_ROOT_PASSWORD=Str0ng!Pass2026
MYSQL_DATABASE=rebar_mgmt
MYSQL_USER=rebar
MYSQL_PASSWORD=Str0ng!Pass2026
```

### 2.4 启动服务

```powershell
cd C:\rebar-mgmt

# 构建镜像并启动所有服务（首次需要下载镜像，约 3-5 分钟）
docker compose up -d

# 查看启动日志（确认无报错）
docker compose logs -f

# Ctrl+C 退出日志查看
```

**启动后验证**：

```powershell
# 检查容器状态（两个容器都应为 Up）
docker compose ps

# 预期输出：
# NAME        STATUS
# rebar-db    Up (healthy)
# rebar-app   Up (healthy)
```

### 2.5 首次访问

```powershell
# 查看服务器 IP
ipconfig

# 浏览器访问
# http://<服务器IP>:5000

# 默认账号：
#   管理员：admin / admin123
#   普通用户：user1 / user123
```

> ⚠️ **安全提醒**：首次登录后，立即修改默认密码！

### 2.6 配置 Windows 防火墙

```powershell
# 以管理员身份运行 PowerShell，开放 5000 端口
New-NetFirewallRule -DisplayName "RebarMgmt Port 5000" `
    -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
```

## 三、方式二：Windows 原生部署

适用于无法使用 Docker 的环境。

### 3.1 安装 Python 环境

```powershell
# 1. 下载 Python 3.9+ (推荐 3.11)
#    https://www.python.org/downloads/
#    安装时勾选 "Add Python to PATH"

# 2. 验证
python --version

# 3. 创建虚拟环境
cd C:\rebar-mgmt
python -m venv venv
venv\Scripts\activate

# 4. 安装依赖
pip install -r requirements.txt
```

### 3.2 安装 MySQL 8.0

```powershell
# 1. 下载并安装 MySQL 8.0 Community Server
#    https://dev.mysql.com/downloads/mysql/
#    安装时设置 root 密码

# 2. 创建数据库和用户
mysql -u root -p
```

```sql
CREATE DATABASE rebar_mgmt CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'rebar'@'localhost' IDENTIFIED BY '<你的密码>';
GRANT ALL PRIVILEGES ON rebar_mgmt.* TO 'rebar'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

```powershell
# 3. 导入初始化数据
mysql -u rebar -p rebar_mgmt < docker-entrypoint-initdb.d\01-init.sql
```

### 3.3 配置并启动

```powershell
cd C:\rebar-mgmt

# 创建 .env 文件
New-Item -Path .env -ItemType File
# 编辑 .env，设置：
#   SECRET_KEY=<随机字符串>
#   DATABASE_URL=mysql+pymysql://rebar:<密码>@localhost:3306/rebar_mgmt?charset=utf8mb4
#   FLASK_ENV=production

# 启动应用
venv\Scripts\activate
python run.py
```

### 3.4 注册为 Windows 服务（开机自启）

```powershell
# 下载 nssm (Non-Sucking Service Manager)
# https://nssm.cc/download

# 注册服务
nssm install RebarMgmt
# GUI 弹窗中设置：
#   Application Path:  C:\Python311\python.exe
#   Arguments:         C:\rebar-mgmt\run.py
#   Startup directory: C:\rebar-mgmt

# 启动服务
nssm start RebarMgmt

# 检查状态
nssm status RebarMgmt
```

## 四、生产环境安全加固

以下配置**必须在对外提供服务前完成**：

### 4.1 修改默认密码

```
□ 修改 admin 账号密码（默认 admin123）
□ 修改 user1 账号密码（默认 user123）
□ 修改 MySQL root 密码
□ 修改 MySQL rebar 用户密码
```

### 4.2 生成安全密钥

```powershell
# .env 中 SECRET_KEY 不能用默认值，否则 session 重启后全部失效
# 用以下命令生成随机密钥：
python -c "import secrets; print(secrets.token_hex(32))"
```

### 4.3 关闭 Debug 模式

`.env` 中确认：
```env
FLASK_ENV=production
```

确认 `config.py` 中 `ProductionConfig.DEBUG = False`。

### 4.4 限制访问 IP（可选）

如果只有公司内部使用，可以限制访问来源：

```powershell
# Windows 防火墙 — 限制 5000 端口只允许特定 IP 段访问
New-NetFirewallRule -DisplayName "RebarMgmt Restricted" `
    -Direction Inbound -Protocol TCP -LocalPort 5000 `
    -RemoteAddress "192.168.0.0/16" -Action Allow
```

### 4.5 配置 HTTPS（推荐）

如果需要外网访问，建议配置反向代理 + HTTPS：

```
用户 → https://域名 → Nginx (HTTPS 终止) → http://localhost:5000 (Flask)
```

*（Nginx 配置详见附录）*

## 五、数据迁移

如果本地已有生产数据，需要迁移到云服务器。

### 5.1 导出本地数据库

```powershell
# 在本地（Docker 环境）
docker compose exec db mysqldump -u root -p rebar_mgmt > backup.sql
# 输入 root 密码

# 或本地 MySQL
mysqldump -u root -p rebar_mgmt > backup.sql
```

### 5.2 上传并导入到云服务器

```powershell
# 将 backup.sql 上传到服务器 C:\rebar-mgmt\

# Docker 环境导入：
docker compose exec -T db mysql -u root -p rebar_mgmt < backup.sql

# 原生 MySQL 导入：
mysql -u rebar -p rebar_mgmt < backup.sql
```

### 5.3 迁移上传文件

```powershell
# 将本地 uploads/ 目录内容复制到服务器 C:\rebar-mgmt\uploads\
```

## 六、日常运维

### 6.1 常用命令速查

```bash
# ===== Docker Compose =====
docker compose up -d          # 启动
docker compose down           # 停止
docker compose restart        # 重启
docker compose logs -f app    # 查看应用日志
docker compose logs -f db     # 查看数据库日志
docker compose ps             # 查看容器状态
docker compose exec app bash  # 进入应用容器
docker compose exec db mysql -u rebar -p rebar_mgmt  # 进入数据库

# ===== 更新应用代码 =====
docker compose down app       # 停止应用容器
docker compose build app      # 重新构建应用镜像
docker compose up -d app      # 启动新版本

# ===== 数据库备份 =====
docker compose exec db mysqldump -u root -p rebar_mgmt > backup_$(Get-Date -Format "yyyyMMdd").sql
```

### 6.2 数据库自动备份

在 Windows 计划任务中创建每日备份任务：

```powershell
# 1. 创建备份脚本 C:\rebar-mgmt\backup.bat
@echo off
set BACKUP_DIR=C:\rebar-mgmt\backups
if not exist %BACKUP_DIR% mkdir %BACKUP_DIR%
set TIMESTAMP=%date:~0,4%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%%time:~6,2%
set TIMESTAMP=%TIMESTAMP: =0%
cd C:\rebar-mgmt
docker compose exec -T db mysqldump -u root -p密码 rebar_mgmt > %BACKUP_DIR%\rebar_mgmt_%TIMESTAMP%.sql
# 删除 30 天前的备份
forfiles /p %BACKUP_DIR% /s /m *.sql /d -30 /c "cmd /c del @path"
```

```powershell
# 2. 创建计划任务（每天凌晨 2 点）
$Action = New-ScheduledTaskAction -Execute "C:\rebar-mgmt\backup.bat"
$Trigger = New-ScheduledTaskTrigger -Daily -At "02:00"
Register-ScheduledTask -TaskName "RebarMgmt Backup" -Action $Action -Trigger $Trigger
```

### 6.3 监控与告警

系统已内置健康检查端点：

```powershell
# 检查服务是否正常
curl http://localhost:5000/health
# 正常返回：{"status":"ok"}

# autoheal 容器会自动重启不健康的容器（Docker Compose 部署已配置）
```

## 七、故障排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 无法访问 5000 端口 | 防火墙未开放 | `New-NetFirewallRule` 开放端口 |
| 容器启动后立即退出 | `.env` 配置错误 | `docker compose logs app` 查看错误 |
| 数据库连接失败 | db 容器未就绪 | 等待 30s 后重试，或 `docker compose restart app` |
| 登录后立即被踢出 | SECRET_KEY 变更 | 确保 `.env` 中 SECRET_KEY 固定不变 |
| 上传文件失败 | uploads/ 目录无写入权限 | `docker compose exec app chmod 777 uploads exports` |
| MySQL 启动失败 | 数据卷权限问题 | `docker compose down -v` 清除数据卷后重新启动 |

## 附录：Nginx 反向代理配置（可选）

如需配置域名 + HTTPS，在服务器上安装 Nginx 后使用以下配置：

```nginx
server {
    listen 80;
    server_name rebar.yourcompany.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name rebar.yourcompany.com;

    ssl_certificate     C:/certs/rebar.pem;
    ssl_certificate_key C:/certs/rebar.key;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
