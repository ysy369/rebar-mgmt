# 健康检查与自动恢复

> 版本：v1.0 | 日期：2026-06-23 | 部署目标：Ubuntu 云服务器 + Docker Compose

---

## 1. 概述

本文档说明钢筋精细化管理平台的健康检查与自动恢复机制，覆盖以下目标：

- **应用层**：Flask 提供 `/health` 端点，真实检查数据库连接与目录可写性；
- **容器层**：Docker healthcheck 检测容器健康，`autoheal` 服务在容器不健康时自动重启；
- **宿主机层**：systemd 定时器每 30 秒运行看门狗脚本，生成 `status/rebar-health.json` 并在异常时留下 `status/rebar-unhealthy.marker` 标记；
- **开机自启**：systemd 服务在服务器启动时自动拉起整个 Docker Compose 平台。

## 2. `/health` 健康检查端点

### 2.1 访问方式

```bash
curl http://localhost:5000/health
```

该端点**无需登录**、**不受 CSRF 保护限制**，供 Docker、负载均衡器、监控脚本调用。

### 2.2 正常响应示例

```json
{
  "status": "healthy",
  "service": "rebar-app",
  "version": "v1.0",
  "timestamp": "2026-06-23T08:30:00+00:00",
  "checks": {
    "database": "ok",
    "upload_folder": "ok",
    "export_folder": "ok"
  }
}
```

### 2.3 异常响应示例

当数据库连接失败或目录不可写时，返回 HTTP 503：

```json
{
  "status": "unhealthy",
  "service": "rebar-app",
  "version": "v1.0",
  "timestamp": "2026-06-23T08:30:05+00:00",
  "checks": {
    "database": "error: ...",
    "upload_folder": "ok",
    "export_folder": "ok"
  }
}
```

### 2.4 代码位置

`app/__init__.py` 中的 `health()` 函数。

## 3. Docker 层自动恢复

### 3.1 容器崩溃自动重启

`docker-compose.yml` 已为 `db`、`app`、`autoheal` 三个服务配置 `restart: unless-stopped`。当容器主进程退出时，Docker 会自动重新创建并启动容器。

### 3.2 容器不健康时自动重启

仅配置 `restart` 不会处理 **healthcheck 失败** 的场景（容器仍在运行，但应用不健康）。因此增加 `autoheal` 服务：

```yaml
services:
  app:
    healthcheck:
      test: ["CMD-SHELL", "curl -fs http://localhost:5000/health || exit 1"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 20s
    labels:
      - autoheal=true

  autoheal:
    image: willfarrell/autoheal:latest
    container_name: rebar-autoheal
    restart: unless-stopped
    environment:
      AUTOHEAL_INTERVAL: 5
      AUTOHEAL_START_PERIOD: 30
      AUTOHEAL_DEFAULT_STOP_TIMEOUT: 10
      AUTOHEAL_ONLY_MONITOR_RUNNING: true
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
```

`autoheal` 每 5 秒扫描一次带 `autoheal=true` 标签的容器；当发现 `rebar-app` 的 `Health.Status` 为 `unhealthy` 时，会执行 `docker restart rebar-app`。

### 3.3 查看容器健康状态

```bash
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Health}}"
docker inspect --format='{{.State.Health.Status}}' rebar-app
```

## 4. 宿主机层：看门狗与状态标记

### 4.1 看门狗脚本

脚本位置：

- `scripts/rebar-health-watchdog.sh`

执行内容：

1. 调用 `http://localhost:5000/health` 获取应用健康状态；
2. 读取 `rebar-app`、`rebar-db` 的 Docker Health.Status；
3. 将结果写入 `status/rebar-health.json`；
4. 若整体不健康，创建 `status/rebar-unhealthy.marker` 并记录到 `logs/health-watchdog.log`；
5. 恢复健康后自动移除 `rebar-unhealthy.marker`；
6. 当环境变量 `RESTART_UNHEALTHY=1` 时，会主动重启 `rebar-app`（与 `autoheal` 互补，默认关闭）。

### 4.2 手动执行

```bash
cd /opt/rebar-mgmt
./scripts/rebar-health-watchdog.sh

cat status/rebar-health.json
cat logs/health-watchdog.log
```

### 4.3 状态文件

| 文件 | 用途 |
|------|------|
| `status/rebar-health.json` | 最近一次健康检查结果 |
| `status/rebar-unhealthy.marker` | 存在即表示平台当前处于不健康状态 |
| `logs/health-watchdog.log` | 看门狗运行日志 |

## 5. 服务器开机自启（systemd）

### 5.1 前提条件

- 项目已部署到 `/opt/rebar-mgmt`（如路径不同，请修改 systemd 单元文件）；
- Docker 与 `docker compose` 命令已可用；
- 具有 root 或 sudo 权限。

### 5.2 安装开机自启

```bash
cd /opt/rebar-mgmt
chmod +x scripts/rebar-startup.sh
chmod +x scripts/rebar-health-watchdog.sh
chmod +x scripts/systemd/install.sh
sudo ./scripts/systemd/install.sh
```

安装脚本会完成：

1. 复制 `rebar-mgmt.service`、`rebar-health-watchdog.service`、`rebar-health-watchdog.timer` 到 `/etc/systemd/system/`；
2. 重新加载 systemd；
3. 启用并立即启动服务与定时器。

### 5.3 相关 systemd 命令

```bash
# 查看平台启动服务状态
sudo systemctl status rebar-mgmt.service

# 查看看门狗定时器状态
sudo systemctl status rebar-health-watchdog.timer

# 查看最近一次看门狗执行日志
sudo systemctl status rebar-health-watchdog.service

# 手动触发平台启动
sudo systemctl start rebar-mgmt.service

# 手动触发一次健康检查
sudo systemctl start rebar-health-watchdog.service
```

### 5.4 服务说明

| 单元文件 | 作用 |
|----------|------|
| `rebar-mgmt.service` | 开机后启动/重启 Docker Compose 平台 |
| `rebar-health-watchdog.service` | 一次性执行健康看门狗 |
| `rebar-health-watchdog.timer` | 每 30 秒触发一次看门狗服务 |

## 7. 常见问题

### Q1：为什么 healthcheck 失败但容器没有重启？

仅 `restart: unless-stopped` 不会响应 healthcheck 失败。请确认 `autoheal` 容器正在运行，且 `rebar-app` 带有 `autoheal=true` 标签。

### Q2：看门狗是否需要 root 权限？

需要。脚本调用 `docker inspect`/`docker restart`，通常只有 docker 用户组或 root 用户有权限。systemd 服务以 root 运行。

### Q3：项目没有放在 `/opt/rebar-mgmt` 怎么办？

修改 `scripts/systemd/*.service` 和 `scripts/systemd/*.timer` 中的 `WorkingDirectory` 与 `ExecStart` 路径，然后重新执行安装脚本。
